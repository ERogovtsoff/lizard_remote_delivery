# Telegram Mini App «Магазин» + бот

## Состав

- **`index.html`** — мини-апп (один файл, открывается из бота)
- **`catalog.json`** — начальный каталог товаров (загружается мини-аппом)
- **`bot.py`** — Telegram-бот на aiogram 3, принимает данные от мини-аппа и пересылает менеджеру

## Что изменилось в этой версии

1. ✅ Удалена опция «Очистить мои данные» из настроек
2. ✅ Убрана валюта ₽, теперь только USD (по умолчанию) и BYN с переключателем
3. ✅ Цены задаются вручную в обеих валютах
4. ✅ Загрузка файлов: фото прикрепляются в мини-аппе, после отправки бот просит клиента переслать их в чат — все фото уходят менеджеру
5. ✅ Сообщения уходят менеджеру @rogovtsoff с упоминанием юзернейма клиента
6. ✅ Админка для управления товарами в «В наличии» и «Идеи» — для @rogovtsoff
7. ✅ Экспорт/импорт `catalog.json`

## Шаг 1. Создание бота

1. Откройте [@BotFather](https://t.me/BotFather), введите `/newbot`, придумайте имя
2. Скопируйте полученный токен (выглядит как `123456:ABC-DEF...`)
3. Там же: `/mybots` → выберите бота → **Bot Settings** → **Menu Button** → введите URL мини-аппа и текст кнопки («Открыть»)

## Шаг 2. Хостинг мини-аппа

Нужен HTTPS. Самые простые бесплатные варианты:

**GitHub Pages** (рекомендую для старта):
1. Создайте новый репозиторий
2. Загрузите туда `index.html` и `catalog.json` в корень
3. Settings → Pages → Source: `main` branch → Save
4. URL получится вида `https://<ваш-логин>.github.io/<репо>/`

**Альтернативы**: Vercel, Netlify, Cloudflare Pages — всё бесплатно.

⚠️ Важно: `index.html` и `catalog.json` должны лежать в одной папке, чтобы мини-апп мог загрузить каталог.

## Шаг 3. Запуск бота

```bash
# Установка
pip install aiogram

# Запуск
export BOT_TOKEN="123:ABC..."
export WEBAPP_URL="https://ваш-логин.github.io/репо/index.html"
export MANAGER_USERNAME="rogovtsoff"   # без @
python bot.py
```

Для production-запуска используйте `systemd`, `supervisor`, `pm2` или Docker — что удобнее. На VPS это запускается одной командой и работает 24/7.

## Шаг 4. Подключение менеджера

Чтобы бот смог отправлять сообщения @rogovtsoff:

1. Менеджер @rogovtsoff открывает бота в Telegram и пишет `/start`
2. Бот ответит «✅ Вы зарегистрированы как менеджер» и сохранит chat_id в файл `manager_chat.txt`
3. С этого момента все запросы и заказы будут приходить ему

**Почему так?** Bot API не позволяет писать пользователю по @username — нужен `chat_id`. Самый простой способ его получить — попросить пользователя один раз написать боту.

## Как это работает

### Запрос на подбор (из главного экрана мини-аппа)

1. Клиент пишет текст / вставляет ссылку / прикрепляет фото и жмёт «Отправить менеджеру»
2. Мини-апп отправляет JSON боту через `WebApp.sendData()` (лимит 4 КБ — без фото)
3. Бот формирует сообщение менеджеру:
   ```
   🆕 Запрос на подбор
   От: @иванов_иван
   ID запроса: 250517183022

   Описание:
   Чёрная толстовка Nike Tech, размер M

   Ссылка: https://...

   📎 Прикреплено фото: 2 — ожидаем пересылку от клиента.
   ```
4. Если есть фото — бот пишет клиенту: «Пришлите фото следующими сообщениями». Все фото, документы и текст, что клиент отправит в течение часа, пересылаются менеджеру с привязкой к ID запроса.

### Заказ из наличия

1. Клиент добавляет товары в корзину и жмёт «Оформить заказ»
2. Мини-апп отправляет JSON со списком позиций, итоговой суммой и валютой
3. Бот пишет менеджеру:
   ```
   🛒 Новый заказ из наличия
   От: @иванов_иван

   Состав заказа:
   • Оверсайз худи бежевый × 1 = $49
   • Карго-брюки чёрные × 2 = $110

   Итого: $159
   ```

### Админка

1. @rogovtsoff открывает мини-апп → Настройки → Админ-панель
2. Добавляет/редактирует/удаляет товары в разделах «В наличии» и «Идеи»
3. Указывает цены отдельно в USD и BYN, размеры через запятую
4. Когда всё готово — вкладка «Экспорт/Импорт» → «Скачать catalog.json»
5. Загружает скачанный файл на хостинг (через GitHub: drag-and-drop в репо, commit)
6. Через пару минут у всех клиентов обновляется каталог

⚠️ Проверка «админ это или нет» — клиентская (по username из Telegram WebApp initData). Этого достаточно, потому что админка только редактирует локальную копию каталога. Реальные изменения становятся видны клиентам только когда вы заливаете обновлённый `catalog.json` на хостинг.

## Файлы, которые создаёт бот при работе

- `manager_chat.txt` — chat_id менеджера (создаётся при `/start` от @rogovtsoff)
- `pending_photos.json` — таблица «клиент → ID запроса» для пересылки фото (TTL 1 час)

Эти файлы безопасны для удаления — при следующем `/start` от менеджера всё восстановится.
