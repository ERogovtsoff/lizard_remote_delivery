# Telegram Mini App: Магазин + заказ из Китая

Мини-приложение для Telegram, открывается через кнопку в боте. Совмещает витрину товаров в наличии и сервис заказа любых товаров из Китая через менеджера.

## Деплой на GitHub Pages

ES-модули требуют HTTPS — `file://` работать не будет.

1. Создайте репозиторий, загрузите все файлы из корня этой папки (`index.html`, `styles.css`, `catalog.json`, `js/`, `db/`)
2. Settings → Pages → Source: `main` / `(root)` → Save
3. Через минуту получите URL вида `https://username.github.io/repo/`
4. В @BotFather: `/setmenubutton` → выберите бота → отправьте URL и подпись кнопки

## Подключение базы данных (Supabase)

По умолчанию работает локальный режим: каталог из `catalog.json`, история и избранное в localStorage. Для синхронизации между устройствами подключите Supabase:

1. Зарегистрируйтесь на [supabase.com](https://supabase.com) и создайте бесплатный проект
2. Откройте **SQL Editor**, скопируйте содержимое `db/schema.sql` и выполните
3. **Project Settings → API**, скопируйте:
   - `Project URL` → `js/config.js` → `SUPABASE_URL`
   - `anon public` ключ → `js/config.js` → `SUPABASE_ANON_KEY`
4. Поменяйте `API_MODE` с `'local'` на `'supabase'`
5. Закоммитьте изменения

**⚠️ Безопасность.** Текущая заглушка Supabase предполагает, что `tg_id` пользователя берётся из `tg.initDataUnsafe.user` — эта строка не подписана и её можно подменить. Это нормально для прототипа, но **перед публичным запуском** нужно:

- Развернуть Supabase **Edge Function**, которая принимает `tg.initData` (подписанную строку), проверяет HMAC-подпись через токен бота и выдаёт JWT с реальным `tg_id`
- Включить **Row Level Security** (политики в конце `schema.sql` — расскомментировать)
- Изменить `js/api/supabase.js`: вызывать сначала Edge Function, потом использовать полученный JWT для запросов

Без этого любой может читать чужие заказы — поэтому пока что в `supabase.js` оставлены заглушки с консольным предупреждением.

## Архитектурные решения

**Почему не React/Vue?** Проект небольшой, размещается на GitHub Pages, не требует сборщика. Vanilla JS + ES-модули дают читаемую структуру без зависимостей. Когда проект перерастёт ~10 000 строк, миграция на React займёт ~1–2 дня.

**Где хранятся настройки (язык/тема/валюта)?** В `customers.preferences` (JSONB) в БД плюс кэш в localStorage. При первом открытии настройки подгружаются с сервера; при изменении пишутся и туда, и туда. Это даёт синхронизацию между устройствами и работу офлайн.

**Где хранится онбординг?** Только в localStorage. Если клиент сменил устройство — нормально снова показать приветствие.

**Где хранится избранное?** В БД (таблица `favorites` с полями `product_id`, `size`). Это даёт синхронизацию и аналитику; одновременно кэшируется в state на клиенте, чтобы UI не моргал.

**Почему в `orders` есть `total_usd` И `total_byn`?** Курс может меняться. Снапшот цен в обеих валютах на момент заказа избавляет от пересчётов.

**Как работает «отправка сообщения менеджеру»?** Через бота (надёжно на всех платформах):

1. Клиент пишет сообщение / оформляет заказ в мини-аппе
2. Мини-апп вызывает `tg.sendData(JSON)` — это официальное API Telegram WebApp
3. Telegram автоматически закрывает мини-апп и отправляет данные боту
4. Бот (`bot.py`) получает их в событии `web_app_data` и пересылает менеджеру

Это работает только если мини-апп открыт через **клавиатурную или inline-кнопку** в боте (что и делает `bot.py` — см. handler `/start`).

Если по какой-то причине `sendData` недоступен (например, апку открыли по прямой ссылке), срабатывает **fallback**: открывается `t.me/@manager?text=...` плюс текст копируется в системный буфер. Клиенту показывается тост-подсказка. На Desktop текст подставляется автоматически; на мобиле клиент делает paste долгим нажатием на поле ввода.

Реализовано в `js/tg.js → sendToBot()`. Файлы прикрепляются клиентом уже в чате с менеджером (мини-апп не умеет передавать файлы боту через WebApp API).

## Запуск бота

```bash
pip install aiogram
export BOT_TOKEN="123:ABC..."           # от @BotFather
export WEBAPP_URL="https://username.github.io/repo/"
export MANAGER_USERNAME="rogovtsoff"
python bot.py
```

**Важно:** менеджер должен один раз отправить боту `/start` — бот запомнит его chat_id в `manager_chat.txt`. Без этого пересылка работать не будет.

После этого мини-апп открывается через кнопку «🛍 Открыть магазин» в чате с ботом (после команды `/start`).

## Локальное тестирование

```bash
cd /path/to/project
python3 -m http.server 8000
```

Затем откройте `http://localhost:8000` (важно — именно через сервер, не как файл). Telegram-функции работают только внутри настоящего Mini App; в браузере можно проверить UI и навигацию.

Для тестирования в Telegram используйте [@BotFather → Test Environment](https://core.telegram.org/bots/webapps#testing-mini-apps) или ngrok.
